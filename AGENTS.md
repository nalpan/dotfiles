# Gitの操作

- コミットメッセージは日本語で記述してください

# GitHubの操作

- GitHubの情報を取得するときは、`gh` コマンドを利用してください。
- Pull Requestを作成する際は必ずdraftにしてください。

# 計画の作り方

原則は以下のようにしてください、別途指示があればその指示を優先してください。

- 計画は作業ディレクトリの `./local-plans` フォルダにMarkdown形式で保存してください。
- 計画には目的が達成されているかの検証作業も含んでください。
- Gitリポジトリで管理されている場合、コミット単位も考慮した計画にしてください。
- 計画のフォーマットは以下の通りにしてください。

```markdown
# 計画のタイトル

計画の概要を簡潔に記載してください。

## 目的

計画の目的を明確に記載してください。

## 実行計画

### Phase 1: フェーズ1のタイトル

- 目的
- 完了条件

#### Step 1: ステップ1のタイトル

- [ ] タスク1
- [ ] タスク2
- [ ] 検証タスク1
- [ ] 検証タスク2

#### Step 2: ステップ2のタイトル

- [ ] タスク1
- [ ] タスク2
- [ ] 検証タスク1
- [ ] 検証タスク2

以後同様にPhaseとStepを追加してください。

```
